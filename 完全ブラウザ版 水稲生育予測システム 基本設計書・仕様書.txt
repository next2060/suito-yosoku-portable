# 完全ブラウザ版 水稲生育予測システム 基本設計書・仕様書 (Final Version)

## 1. プロジェクト概要
本システムは、インターネット接続が不安定な現場や、厳格なセキュリティポリシー（exe実行不可）環境下での利用を想定した、完全クライアントサイド動作の水稲生育予測アプリケーションである。
Webブラウザの機能のみでローカルファイルの読み書きを行い、地図表示・属性管理・生育シミュレーション・レポート出力を行う。
背景地図のみインターネット（国土地理院サーバー）から取得する「ハイブリッド構成」を採用し、配布容量の軽量化と実用性を両立させる。

---

## 2. システム要件

### 2.1 動作環境
* **OS:** Windows 10 / 11 (64bit推奨)
* **ブラウザ:** Google Chrome (最新版), Microsoft Edge (Chromium版)
* **ネットワーク:**
    * **地図表示時:** インターネット接続推奨（地理院タイル取得のため）。
    * **データ処理時:** 完全オフライン可（機密データは外部送信されない）。
* **インストール:** 不要（フォルダ配置のみ）。

### 2.2 技術スタック (Client-Side Only)
| 領域 | 採用技術 | 用途 |
| :--- | :--- | :--- |
| **アプリ基盤** | **Next.js (Static Export)** | 静的HTMLとして出力し、サーバーレスで動作。 |
| **地図表示** | **React Leaflet** | 地理院地図タイルの表示。 |
| **データ操作** | **File System Access API** | ローカルフォルダ（CSV/JSON/GeoJSON）の読み書き。 |
| **計算ロジック** | **TypeScript** | 生育シミュレーション、気象データ結合処理。 |
| **日長計算** | **suncalc** | 緯度経度と日付から日長を算出（バンドル同梱）。 |

## 4. データ構造仕様

### 4.1 気象データ (CSV)
各地点につき「実測値(_act)」と「平年値(_avg)」の2ファイルをペアで配置する。
* **命名規則:** `{地点名}_act.csv`, `{地点名}_avg.csv`
* **優先順位:** 日付ごとにデータ有無を確認し、実測値を最優先する。欠測時および未来の日付は平年値を使用する。

### 4.2 ユーザー品種マスタ (JSON)
標準品種をベースに、ユーザーが補正値を設定した品種。
* **構成要素:**
    * `base_variety`: ベースとなる標準品種ID（例: koshihikari）
    * `heading_offset`: 出穂日の補正日数（例: +2, -3）
    * `maturation_temp`: 独自の登熟積算気温（℃）

### 4.3 圃場登録DB (JSON)
筆ポリゴンのIDをキーとして属性を管理する。
* **構成要素:** ID, 圃場名, 品種ID, 移植日, 出穂実績日(任意)

## 5. 機能詳細仕様

### 5.1 初期化・データロード
1.  **フォルダアクセス:** アプリ起動時、ユーザーに `data_folder` の選択とアクセス許可を求める。
2.  **インデックス生成:** `fields`, `weather` フォルダを走査し、選択リスト（市町村、気象地点）を自動生成する。
3.  **DBロード:** `user_db.json`, `user_varieties.json` をメモリ上に展開する。

### 5.2 地図表示・操作
1.  **背景地図:** 地理院タイル（標準/オルソ）を表示。ブラウザキャッシュを活用し、直近の表示エリアはオフラインでも閲覧可能とする。
2.  **圃場描画:**
    * 選択された市町村のGeoJSONをロード。
    * `user_db.json` と照合し、「登録済み（緑）」と「未登録（グレー）」に色分けして表示。

### 5.3 品種管理機能
1.  **標準品種（内蔵）:** 一番星, あきたこまち, ふくまるSL, コシヒカリ, にじのきらめき, 笑みたわわ。
2.  **カスタム登録:** ユーザーは標準品種を継承し、出穂補正と登熟積算気温を設定して新規品種を作成・保存できる。

### 5.4 圃場情報登録
1.  **入力:** 地図上のポリゴンを選択し、品種（標準/カスタム）、移植日、出穂実績（任意）を入力して保存。
2.  **保存:** `user_db.json` を即時更新し、地図上のステータスを変更する。

### 5.5 生育予測ロジック
選択された気象地点の合成データ（Act優先）を使用する。

* **パターンA（出穂実績なし）:**
    * **出穂予測:** 移植日から開始。DVSモデルにより出穂日を計算（品種補正を加味）。
    * **成熟予測:** 予測出穂日を起点に、登熟積算気温を満たす日を計算。
* **パターンB（出穂実績あり）:**
    * **出穂日:** ユーザー入力値を確定値とする。
    * **成熟予測:** 実績出穂日を起点に、登熟積算気温を満たす日を計算（高精度）。

### 5.6 成果物の出力（個別選択方式）
用途に合わせて以下のファイルを生成・ダウンロードする。

1.  **レポート出力 (HTML):** 現在の画面範囲の地図画像をBase64埋め込みした、単独閲覧可能なHTMLファイル。
2.  **GISデータ出力 (GeoJSON):** 登録済み圃場の形状と、予測結果属性を含むGIS用データ。
3.  **集計リスト出力 (CSV):** 登録済み圃場の予測結果一覧（BOM付きUTF-8）。

## 6. 非機能要件

* **セキュリティ:** ユーザーデータはローカルディスク内に留まり、外部送信されないことを保証する。
* **パフォーマンス:** 44市町村のデータを分割管理し、必要なデータのみメモリにロードすることでブラウザのクラッシュを防ぐ。
* **容量:** GeoJSONの軽量化（属性削除・座標丸め）を行い、配布パッケージサイズを抑制する。
