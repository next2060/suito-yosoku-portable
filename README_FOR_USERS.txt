水稲生育予測システム（ポータブル版） 取扱説明書

============================================================
１ システム概要
============================================================
本システムは、インターネットブラウザ上で動作する水稲生育予測アプリケーションです。
外部のクラウドサーバー等を介さず、お使いのPCローカル環境内でデータ処理・保存が完結するため、
機密性の高い圃場データ等を安全に管理しながら、生育予測計算や衛星画像（NDVI/冠水状況）の確認を行うことができます。
（※地図の基本描画や、事前のデータ取得にはインターネット接続が必要です）

============================================================
２ ディレクトリ構成
============================================================
配布されたZIPフォルダ（suito-yosoku-portable）を展開すると、以下の構成になっています。

suito-yosoku-portable/
│
├─ index.html           : ★アプリ本体の起動ファイル（ブラウザで開きます）
├─ README_FOR_USERS.txt : この説明書ファイル
│
├─ data_folder/         : すべてのデータを保存する最重要フォルダ
│   ├─ user_db/         : (自動生成) 登録した圃場データが保存されます (user_db.json)
│   ├─ fields/          : 各市町村の圃場区画ポリゴンデータ (geojson) が格納されます
│   ├─ weather/         : アメダスからの気象データ(CSV)が格納されます
│   │   ├─ 〇〇2026_act.csv : 今年の観測実績データ
│   │   └─ 〇〇2026_avg.csv : 過去の平年値データ
│   ├─ satellite/       : 外部から取得した衛星画像データや解析結果
│   │   ├─ TrueColor/   : 衛星の自然色写真データ (tiff)
│   │   ├─ NDVI/        : 植生活性度の解析データ (json/tiff)
│   │   ├─ Flooded/     : 光学衛星に基づく冠水状況の解析データ (json/tiff)
│   │   └─ SAR_Flooded/ : SAR（合成開口レーダ）に基づく冠水状況の解析データ (json/tiff)
│   └─ varieties.json   : 登録した品種の計算パラメータが保存されます
│
├─ DataFetcher/         : 外部データを自動取得するためのプログラム群
│   ├─ WPy64-31241/     : Python実行環境 (WinPython)
│   ├─ update_amedas.bat   : ★気象データを最新化する実行ファイル（ダブルクリック）
│   ├─ update_sentinel.bat : ★衛星データを最新化する実行ファイル（ダブルクリック）
│   ├─ get_amedas.py    : (内部処理用) 気象データを取得するスクリプト本体
│   └─ get_sentinel.py  : (内部処理用) 衛星データを取得するスクリプト本体
│
└─ src/ 等              : アプリケーションを動かすためのシステムファイル群（通常は編集不要）

============================================================
３ データの準備（DataFetcherの使用方法）
============================================================
本アプリで正確な予測や最新の衛星画像を見るためには、定期的に「気象データ」と「衛星データ」を
外部（気象庁・Planetary Computer等）からダウンロードし、data_folder 内を更新する必要があります。
同封されている WinPython を使用するため、複雑な環境構築なしで実行可能です。

【実行手順】
1. `DataFetcher` フォルダを開きます。
2. 以下のバッチファイル（`.bat`）をダブルクリックして実行します。
   - 気象データの更新: `update_amedas.bat` を実行
   - 衛星データの更新: `update_sentinel.bat` を実行
   （※黒い画面が立ち上がり、自動でデータ取得が始まります。完了するまで閉じずにお待ちください）
3. 実行が完了すると、自動的に `data_folder` 内の所定のフォルダへ最新のCSVやTIFF画像が保存されます。

※注意: 庁内回線などプロキシ（Proxy）環境外で実行する場合は、スクリプト実行前にプロキシ設定が必要になる場合があります（後述の「６ Ｑ＆Ａ」を参照）。

============================================================
４ アプリの起動
============================================================
【手順】
1. フォルダの直下にある `index.html` をダブルクリックしてブラウザで開きます。
   （推奨ブラウザ: Google Chrome, Microsoft Edge）※Internet Explorer非対応
2. 起動直後の画面で「Select Data Folder（データフォルダを選択）」ボタンをクリックします。
3. ダイアログが開いたら、同梱されている `data_folder` のフォルダ自体を選択し、「アップロード」または「選択」をクリックします。
4. ブラウザが「このサイトに...のファイルの表示を許可しますか？」と尋ねてくる場合は、「表示（または変更の保存）を許可」してください。

※セキュリティ上の仕様により、ブラウザの「再読み込み（F5）」を行うとフォルダへのアクセス許可がリセットされます。その場合は手順2から再度フォルダを選択し直してください。

============================================================
５ 各ページの使い方
============================================================

------------------------------------------------------------
5-1. MAP（メイン画面）
------------------------------------------------------------
地図を見ながら、直感的に圃場の管理と予測を行うメイン画面です。

- **市町村・アメダス地点選択**: 画面左側のパネル（Setupタブ）から、対象の市町村と、予測に使用するアメダス観測所を選択・変更できます。
- **圃場情報の入力・保存**:
  1. 地図上で圃場にしたい場所をクリックすると、青枠で選択されます。
  2. 左側の「Field Data」タブに、選択した圃場の詳細情報（圃場名、品種、田植え日など）を入力するフォームが現れます。
  3. 日付は「0501」のように入力しても、自動で「05-01」に整形されます。
  4. 入力後「Save (Update)」ボタンを押すと、予測結果が計算され地図上にポップアップ（バルーン）で表示・保存されます。
- **状態フラグ（実績 / 予測）の見方**:
  圃場をクリックした際のポップアップや、左側のサイドメニューには日付の横に（実績）や（予測）というフラグが付きます。
  * `(実績)`: ユーザーが手動で直接入力した「確定済み」の過去データであることを示します。
  * `(予測)`: システムが気象データ（アメダス）と品種パラメータから自動計算した予測データであることを示します。
  例えば「田植え日」と「品種」だけを入力して Save した場合、その後の出穂・成熟はすべて `(予測)` として自動計算されます。
  逆に、実際に出穂した日付を手動で上書きすると、そこが `(実績)` に変わり、以降の成熟期のみが再計算されて `(予測)` になります。
- **圃場の削除**: 登録済みの圃場を選択し、「Delete」ボタンを押すことで user_db.json から削除されます。
- **ファイル出力（Exportタブ）**: 選択した圃場データを、CSV・HTML・geojson形式で出力できます。

------------------------------------------------------------
5-2. WEATHER（気象データ確認画面）
------------------------------------------------------------
予測計算の根拠となっている「アメダスの気温・降水量」の推移を、グラフで詳細に確認できます。

- **観測所の選択**:
  画面上部の「アメダス観測所」プルダウンから、表示したい観測所を選びます。
- **グラフの仕様（実績値と平年値）**: 
  1つのグラフの中に、以下の8系列が重ねて表示されます（チェックボックスでON/OFF可能）。
  ① 平均気温(実) / 平均気温(平)  
  ② 最高気温(実) / 最高気温(平)  
  ③ 最低気温(実) / 最低気温(平)  
  ④ 降水量(実)   / 降水量(平)
- **データの読み方**:
  横軸は日付（MM-DD）、左縦軸は温度(℃)、右縦軸は降水量(mm)を示します。マウスオーバーすると、その日の各値がツールチップで表示されます。
- **印刷・資料用の画像出力**:
  右上の「保存用サイズに固定」にチェックを入れると、A4縦向き想定の固定レイアウト(1200×900px)で表示されます。
  その状態で「画像を保存」ボタンを押すと、現在表示中のグラフをPNG画像としてダウンロードできます（報告書や資料への貼り付けに利用できます）。

------------------------------------------------------------
5-3. CSV IMPORT（データ一括取り込み画面）
------------------------------------------------------------
Excelで管理・編集した圃場リスト（CSV形式）を、一括でアプリに取り込む画面です。

- **基本的な操作の流れ**:
  1. 画面左パネルで「1. Select City」から市町村を選択します。  
  2. 「2. Target DB」で、取り込み先となるデータベース（DB）名を選択します。  
  3. 「3. Upload CSV」でCSVファイルを選択すると、ポリゴンUUIDと照合して自動で圃場がマッピングされます。  
  4. 右側の地図上に、CSV内のUUIDと一致した圃場だけがハイライト表示されます。  
  5. 「Save To DB」ボタンを押すと、マッチした圃場情報が選択中のDBに保存されます。
- **対応している主なCSV列**:
  - ポリゴンID: `polygon_uuid` / `id` / `fid` / `ポリゴンUUID`  
  - 圃場名: `name` / `field_name` / `圃場名`  
  - 品種: `varietyId` / `variety` / `品種`  
  - 日付: `transplantDate` / `headingDate` / `maturityDate`  
    （日本語列名 `移植期` / `移植日` / `出穂期` / `成熟期` も読み込み可能）  
  - 状態フラグ: `headingStatus` / `maturityStatus` / `出穂期_状態` / `成熟期_状態`
- **安全性（不正値の取り扱い）**:
  取り込むCSVに「計算エラー」などの不正な文字列が混ざっていても、日付以外の不正な文字は自動的に空欄として弾かれるため、安全に取り込めます。日付列についても、形式チェックと簡易的な自動整形を行います。
- **保存後の一括予測 (Re-Predict Maturity)**:
  CSVインポートおよびDB保存後、画面下部の「Re-Predict Maturity」ボタンを押すことで、選択中の気象データを用いて、取り込んだ「全圃場の成熟期」を一括で再計算し、上書き保存します。  
  （※すでに出穂期が「実績データ」として確定している圃場に対してのみ予測計算が行われます。ボタンを有効化するには、先に気象データを選択・読み込みしておく必要があります）

------------------------------------------------------------
5-4. SATELLITE（衛星画像ビューアー）
------------------------------------------------------------
取得したSentinel衛星データをもとに、農地の状況を視覚的に分析する画面です。

- **TrueColorモード**: 衛星が撮影したそのままの写真（肉眼に近い色）を地図上に重ねて表示します。雲の有無や全体的な様子を大まかに把握するのに適しています。
- **NDVIモード**: 植生の活性度（生育状況）を解析したポリゴンを表示します。0(白)から1(濃い緑)のグラデーションで色が塗られ、緑が濃いほど生育が盛んであることを示します。
- **Floodedモード**: 圃場の冠水（水没）状況を解析したポリゴンを表示します。光学衛星に基づく MNDWI（改良型正規化水指数）と、SAR（合成開口レーダ）に基づく 2 種類の冠水判定に対応しており、冠水判定が出ている箇所は青く、そうでない箇所はオレンジで表示されます。ツールチップ（クリック時）で、手法ごとの詳細な冠水割合を確認できます。※NDVIおよびFloodedモード中は、処理を軽くするため衛星写真自体は非表示になり、色付きのポリゴンのみが表示される仕様になっています。

------------------------------------------------------------
5-5. Manage Varieties（品種設定画面）
------------------------------------------------------------
予測計算に使用する「品種」の追加や、パラメータの微調整を行う裏設定画面です。

- **品種の追加**: 従来の複雑なID入力が不要になり、「品種名（Name）」を入力するだけで新規登録が可能です。
- **基準品種からのコピー**: 新しい品種を作る際、「基準品種」のドロップダウンから似ている既存品種（例：コシヒカリ）を選ぶと、複雑な生長パラメータがすべて自動コピーされます。
- **出穂期のオフセット (`Adj`)**: 基準品種より「何日早く出穂するか（または遅いか）」を日数（例：-3, +5）で入力することで、簡易的に計算モデルをズラすことができます。
- **成熟期積算気温 (`tmax`)**: その品種が出穂してから成熟するまでに必要とする積算気温（℃）を設定します。

============================================================
６ よくある質問・注意事項
============================================================

Q. ページをリロード（再読み込み）したらデータが消えた！
A. セキュリティ上、再読み込みをすると `data_folder` への接続が切断されます。
   画面の指示に従い、再度 `data_folder` を選択し直すと、保存されていたデータが元通り表示されます。

Q. 衛星画像（Satellite）を開いても何も表示されない
A. `DataFetcher`フォルダ内の `get_sentinel.py` を実行して、事前に衛星データを取得している必要があります。
   また、該当する日付・市町村に雲がかかっていて有効なデータが取得できていない場合も表示されません。

Q. 品種設定で既存の品種の名前を変更したい
A. データの整合性を保つため、一度作成した品種の名前（ID）は変更できません。
   間違えた場合は、お手数ですが一度「Delete」で削除してから再度作成し直してください。
